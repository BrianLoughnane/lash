# Permission Sets

##### Manage what users are permitted to view and edit across Atrius

Manage what users are permitted to view and edit across Atrius.

A permission set is a collection of features and capabilities with prescribed authorization levels for users. Every user is assigned to a single permission set.

To view permission sets, go to Settings on the left side menu. Select User Management, then go to the Permission Sets tab.

**Default permission sets**

Default permission sets (Administrator, Editor, and View-Only User) are preconfigured and uneditable, while custom permission sets can be configured by an organization's administrators.

*Administrators* have edit access to the majority of your organization's licensed features and capabilities. There should be at least one Administrator within your organization. However, it is recommend that you limit the number of Administrators only to those who need this broad-based access.

*Editors* have edit access to the majority of your organization's licensed features and capabilities, but cannot access Organization Settings and certain data settings.

*View-Only Users* have view-only access to the majority of the licensed features and capabilities available to Editors. However, these users are prohibited from performing data editing operations, managing or adding buildings or points, creating or editing dashboards, or managing or inviting users. View-Only Users can still download data and receive email notifications.

**How to read the permission tables**

If a default permission set is granted access to a feature or capability, then it is denoted by a green checkmark. If none of the three default permission sets have access, then that feature may be accessed only by a Partner Administrator or Customer Support.

**Custom permission sets**

If default permission sets do not meet your requirements, then you may add a custom permission set. Only Administrators may do so.

On the Permission Sets tab, select the 'Add a permission set' button.
