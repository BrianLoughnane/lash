# Notifications

Monitor the status of points and bill points based on notification thresholds that you define.

#### 👍 This article will help you:

* Configure an appropriate offline threshold or flatline threshold for a point
* Receive email notifications when a point status has changed
* Find integration-specific troubleshooting resources

To access Notifications, select the Notifications bell icon at the top of any page.

#### Notifications table

The main Notifications page displays a table of all notifications generated for each of your points and bill points. More recent notifications display at the top of the table. Use the Filter to narrow your selection of notification types to view.

#### Point status

A point or bill point's status is determined by the presence of data flow, based on its offline threshold. If a point or bill point does not receive data for a period of time that exceeds the offline threshold, then its status will change to 'Offline' or 'Outdated', respectively.

To view the status of all points or bill points, go to Points or Bills on the left side menu.

| Status | Description |
| --- | --- |
| Online | The point is actively receiving data. |
| Offline | The point has not received data for a period of time longer than the offline threshold. |
| Flatlined | The point has been receiving a repeating value for a period of time longer than the flatline threshold. |
| Current | The bill point is actively receiving data. |
| Outdated | The bill point has not received data for a period of time longer than the offline threshold. |
| Historical | The point or bill point is not expected to receive any new incoming readings. |
