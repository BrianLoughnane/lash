# Analytics, Alerting, & Project management

#### Tasks

Updated over a week ago

##### Make better, data-driven decisions about your tasks

Tasks allows you to create and manage to-do lists and data collection for reporting cycles and portfolio maintenance.

To access Tasks, go to Apps in the side menu at left, then Tasks.

**Overview**

Create action-oriented task lists related to upcoming reporting deadlines or ongoing data maintenance. Streamline data collection for reporting by enabling direct data submission from stakeholders, create reminders for efficient task management, and monitor task completion all in one place.

The homepage displays all tasks in an organization:

* Progress bar: Displays progress (%) of completed tasks against all tasks in the filtered view
* Search: by task name, assignee or approver
* Filter: by status, task type, assignee, or approver
* Set period: Filter by task Due Date
* Add task: Create a new data collection or other type task. Select the dropdown to add a task series.
* Edit task: Click to edit an existing task
* Duplicate task: Click to duplicate an existing task, and update fields for quicker task creation
* Delete task: Permanently delete a task record

**Status**
* Pending: Task has been created, and will be sent once the Issue date occurs
* Sent: Task has been sent to the assignee via email on the Issue date, and is waiting to be completed
* Ready for review: Data collection task has been completed by the assignee and is waiting to be reviewed.
* Completed: Other type task has been Completed by the Assignee

**Add a task**

Create a single task to collect data or set a reminder. Enter the Task name, select a Task type, and set the Issue date and Due date.

**Task types**

*Other*: Other tasks are intended to capture to-do list items that will help you better manage your projects.

*Data collection*: Data collection tasks facilitate the collection of manual data for hard to automate data sources, such as Scope 3 data, utility data in leased spaces, and general data gaps.

#### Alerts

Updated over 2 weeks ago

##### Automated alerting to help you proactively manage your portfolio

Alerts allows you to automatically notify users when a specific condition is detected in a building, giving teams the information they need to take immediate action.

To access Alerts, go to Apps on the left side menu, then Alerts.

**Overview**

Alerts can be configured for individual points and bill points—including calculated points and bill points.

There are three ways to add an alert:
1. Go to the Alerts app.
2. Go to the Alerts tab for any point or bill point.
3. Go to Points or Bills on the left side menu. Select one or more points or bill points using the checkboxes on the left side of the table. Then, go to Actions > Create alert.

**Alert Configuration Fields:**
- Name: The alert's name. Each name should be easily distinguishable from other alert names.
- Data type: Select a data type to filter the list of available point types displayed below.
- Point type: Select a point type to filter the list of available points displayed below.
- Points: The point(s), bill point(s), or sensor(s) to which the alert pertains.
- Condition: The condition that triggers the alert.
- Unit: The unit for the corresponding condition's value.
- Value: The value for the corresponding condition.
- Month: The month(s) during which the alert applies.
- Day of week: The day(s) during which the alert applies.
- Start / End: The time of day during which the alert applies.
- Recipients: The Atrius users who are notified by email when an alert is generated.
- Message (optional): A description, instructions, or note to be displayed with notifications.
- Delivery frequency: The interval over which the alert will be delivered.

#### Projects

Updated over 3 months ago

##### Projects quantifies the results of energy efficiency investments performance improvements using industry-standard verification protocols

Projects quantifies the results of energy efficiency investments and other building performance improvements using industry-standard verification protocols.

To access Projects, go to Apps in the side menu at left, then Projects.

**Add a project**

To add projects, select the 'Add a project' button. The Projects table displays all the projects at your organization. Total savings opportunity shows the estimated cost savings of all projects in the current table view, while Savings implemented so far shows the calculated cost savings of all projects in the current table view that are 'In progress' or 'Completed'.

**Project Status:**
- Pre-seed: To do
- Baseline seed: To do
- Implementation: In progress
- Performance: In progress
- Post-performance: Completed

**How is savings calculated before and after implementation?**

When the performance period is in effect, the estimated values you provide for 'Estimated annual resource savings' and 'Estimated payback period' on a project's Profile are overridden by actual, calculated values, based on the points and baselines associated with the project.

**Dashboards**

A standard dashboard is automatically generated by Atrius for each project. Any number of custom dashboards may also be added to a project.

**Profile**

A project includes a mandatory baseline seed period, implementation period, and performance period as well as optional cost and project detail information.

#### Heat Map Analysis

Updated over a year ago

##### Heat Map Analysis allows you to find energy efficiency improvement opportunities by displaying a year of hourly data in a single graph

Heat Map Analysis allows you to find energy efficiency and building scheduling improvement opportunities by displaying an entire year of hourly data in a single graph.

To access Heat Map Analysis, go to Apps in the side menu at left, then Heat Map Analysis.

**Overview**

Heat Map Analysis is a color-coded grid, consisting of 24 hours on the x-axis and a specified number of days on the y-axis, that displays average demand for a selected point. It can plot up to an entire year of hourly data: 8,760 data points.

Cell color represents relative average demand during each one-hour period. For consumption points, red indicates very high consumption, and dark green indicates very low consumption. For production points, dark green represents high energy generation, and light green represents low energy generation.

**Configuration Fields:**
- Building: Select a building.
- Point: Select the data point you wish to graph.
- Period: Select the time period. Choose from standard calendar views or rolling views.
- Overlay (optional): Select an optional graph overlay, such as degree days, outdoor temperature, peak demand, or baseload.

**Best practices & ways to use**

*Determine if usage correlates with occupancy*: For most resources, usage will correlate with occupancy if the building is well-scheduled and well-managed.

*Determine if usage correlates with outdoor temperature*: In many buildings, resource use will correlate with outdoor temperature and weather conditions in addition to correlating with occupancy.

#### Load Profile Analysis

Updated over 2 years ago

##### Load Profile Analysis allows you to quickly identify load anomalies by plotting interval data over a daily time series for specific days

Load Profile Analysis allows you to quickly identify load anomalies by plotting interval data over a daily time series for specific days of the week.

To access Load Profile Analysis, go to Apps in the side menu at left, then Load Profile Analysis.

**Overview**

In the Load Profile graph, you can configure:
- Building: Select a building.
- Point: Select the data point you wish to graph.
- Period: Select the time period. Choose from standard calendar views or rolling views.
- Days: Select a day view, such as weekends, weekdays, or a particular day of the week.
- Overlay (optional): Select an optional graph overlay. Choose from daily average temperature or daily peak temperature.

The Max (red) trend line is the profile with the highest overall peak demand value. The Min (green) trend line is the profile with the lowest overall peak demand value. The Average (orange) trend line that is an average of all the profile lines.

**Best practices & ways to use**

- Determine if your buildings have scheduling in place
- Find deviations in your building schedule
- Identify and curtail peak demand use
- Isolate the effects of weather on building performance

#### Trend Analysis

Updated over a year ago

##### Trend Analysis allows you to quickly graph consumption or interval demand for a building

Trend Analysis allows you to quickly graph consumption or interval demand for a building.

To access Trend Analysis, go to Apps in the side menu at left, then Trend Analysis.

**Overview**

In the Trends graph, you can configure:
- Building: Select a building.
- Point: Select the data point you wish to graph.
- Period: Select the time period. Choose from standard calendar views, rolling views, or a custom period.
- Overlay (optional): Select an optional graph overlay, such as previous period, outdoor temperature, baseline, or HDD/CDD.

Below the graph, several statistics are displayed in one of three types of tables. Each table is automatically selected depending on the graph configuration:
- General statistics (default) displays consumption, cost, and demand data.
- Utility Rate & Tariff Engine statistics displays general statistics, plus demand cost data.
- Baseline statistics displays general statistics, plus baseline data.

**Notes on the Trend Analysis app**
- At periods of 7 days or less, data will be plotted at 1-, 5-, or 15-minute intervals
- At periods above 7 days in length, the graph automatically changes from line (demand) to bars (consumption/production)
- HDD, CDD, and Baseline overlays only render for periods of 7 days or more

**Best practices & ways to use**
- Analyze data quality and point uptime
- Identify anomalies
- Compare year-over-year changes in consumption
- Determine when a demand peak was hit
- Understand how external factors influence resource use

#### Portfolio Drift

Updated over 2 weeks ago

##### Portfolio Drift provides a portfolio-wide view of changes in your buildings' performance, helping you focus on high-priority buildings

Portfolio Drift provides a portfolio-wide view of changes in your buildings' performance, helping you to focus attention on high-priority buildings.

To access Portfolio Drift, go to Apps in the side menu at left, then Portfolio Drift.

**Overview**

Portfolio Drift is a "tree map" graph that can be useful for showing how individual buildings contribute to the overall performance of your portfolio, and for tracking how building performance changes—or drifts—over time. The graph displays the quantity of a metric via the size of a cell, and percent change from the previous period via the color of a cell.

**Configuration Fields:**
- Category: Select a building group or "all buildings".
- Point type: Select a point type you wish to graph, such as electricity, natural gas, or water.
- Metric: Select the metric, such as mean, peak demand, baseload, or EUI.
- Period: Select the time period. Choose from standard calendar views or rolling views.
- Overlay: Select a previous period overlay.

**EUI calculation**

EUI is expressed in units of kBTU/ft^2, and is computed by assuming that the rate during the selected period is consistent for an entire year.

**Best practices & ways to use**

Managers of large, diverse building portfolios have to overcome a lot of challenges. Portfolio Drift helps answer these questions by highlighting facilities that have deviated from their historical consumption patterns and emphasizing those with the largest impact on your portfolio.

*Understand consumption patterns*: Changes in consumption patterns are not necessarily a bad thing, and changes can happen for predictable and expected reasons.

*Identify deviations*: Once an optimized schedule is established, chances are you want it to stay that way. Portfolio Drift highlights individual buildings with the largest percent change in certain key metrics.
